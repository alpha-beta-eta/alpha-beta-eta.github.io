第1章 合约

第1.1节 幻灯片版本

此课开头的任务是理解以下代码做了什么:

int f(int x, int y) {
  int r = 1;
  while (y > 1) {
    if (y % 2 == 1) {
    r = x * r;
    }
    x = x * x;
    y = y / 2;
  }
  return r * x;
}
当然了, 本质上我们需要先明确我们需要做什么. 对于该课程, 从技术性角度来说, 需要做的有以下三件事情:
  • 确定前条件,
  • 确定后条件,
  • 证明前条件可以推出后条件, 可能需要添加循环不变量之类的断言以辅助证明.
这也可以笼统地表达为证明程序的正确性.

抛开课程故意做作的演进过程, 我们立即可以看出这个函数可以计算输入的幂. 然而, 显然它也没有考虑一些边角情况. 不过, 我们也可以在前条件里就将边角排除. 当然, 绝大多数时候, 我们还是希望尽可能一般.

我们的思维方式如下: 设该函数的输入分别是x0和y0, 循环不变量是r⋅xy=x0y0. 如果y0是满足y0≥1的整数, 那么在离开while循环时y的值应该为1, 而x1=x. 因此, 在离开while循环时, 我们有r⋅x=x0y0. 这个推理几乎是完全正确的, 不过并没有考虑一些边角情形, 尤其是和带符号整数大小限制相关的. 当然了, 按照课程安排, 这个在之后的讲座才会考虑. 不过, 以上我们至少注意到了如果y0=0, 那么推理就会被破坏而失效.

此课最终给出的版本如下:

int f(int x, int y)
//@requires y >= 0;
//@ensures \result == POW(x,y);
{
  int b = x;
  int e = y;
  int r = 1;
  while (e > 0)
  //@loop_invariant e >= 0;
  //@loop_invariant POW(b,e) * r == POW(x,y);
  {
    if (e % 2 == 1) {
      r = b * r;
    }
    b = b * b;
    e = e / 2;
  }
  //@assert e == 0;
  return r;
}
这实际上是C0语言而非C语言的代码.
