余代数引论

前言

数学是研究关于计数, 测量, 变换等领域潜在的形式结构的学科. 它发展出了诸如数系, 群, 向量空间 (参见例如[345]) 等基本概念, 并对其性质进行了深入研究. 近几十年来, 动态特征也逐渐成为研究的对象. 计算机的出现推动了这一发展. 动态性通常涉及某种事态, 这种事态可以被观察和修改. 例如, 图灵机纸带上的内容构成了其状态的一部分. 因此, 这样的机器可能拥有许多可能的状态, 并能从一个状态转移到另一个状态. 类似地, 计算机所有存储单元的内容总和也可以被理解为计算机的状态. 用户可以通过屏幕 (或打印机) 观察该状态的一部分, 并通过输入命令来修改这一状态. 作为响应, 计算机会表现出特定的行为. 描述这样的一种计算机系统的行为并非平凡的事务. 然而, 如果我们希望对系统的行为进行形式化推理, 就需要对此类复杂系统作出形式化描述. 这种推理对于确保系统的正确性或安全性是必要的. 它包括一份描述所需行为的规约, 以及一个刻画给定实现满足该规约的正确性证明.

数学家和计算机科学家都引入了各种用以捕获基于状态的动态性的本质的形式结构, 例如(各种形式的)自动机, 转换系统, Petri网和事件系统. 余代数这一领域在理论计算机科学内部兴起, 并提出了统一的主张. 它旨在成为计算性动态的数学. 它结合了来源于动力系统的数学理论和基于状态的计算理论的概念和想法. 余代数领域目前仍处于起步阶段, 但有望通过提供适当的语义基础及配套逻辑, 将微分方程理论与自动机理论, 进程理论以及生物计算和量子计算统一起来. 余代数理论可被视为源自理论计算机科学领域的原创性贡献之一. 余代数的应用范围目前仍相当有限, 但未来有望扩展至物理, 生物或经济学等领域中的动态现象——例如, 基于Adleman (DNA计算之父) 所提出的生物生命可等同于计算的主张[32]; 基于[362]对经济学中所用类型空间的余代数描述[211]; 基于[52]对上述所有领域共有的网络动态性的描述; 基于[447]在生物建模中对余代数的运用; 以及基于[7, 249]在量子计算中引入余代数的工作.

余代数具有令人惊讶的简单性. 它们由一个状态空间 (或状态的集合), 设为X, 连带着一个具有形式X→F⁡(X)的结构映射构成. 符号F描述了某种牵涉X的表达式 (F是一个函子), 其捕获了结构映射应用于状态的可能结果. 映射X→F⁡(X)以作用于状态上的函数之形式捕获了动态性. 例如, 对于非确定性计算, 我们可令F为幂集, 即F⁡(X)=𝒫⁡(X), 而对于非终止计算我们可置F⁡(X)={⊥}∪X. 在达到了这种一般性的层次上, 代数被描述为余代数的对偶 (反之亦然), 即作为具有形式F⁡(X)→X的映射. 这一抽象观念的诱人之处在于结构 (代数) 和行为 (余代数) 之间的对偶.

计算机科学是关于被生成行为的

计算的本质是什么? 计算机科学这一学科的主题是什么? 常见的回答是数据处理或符号操作. 在此, 我们采用一种更具行为主义色彩的方法, 将计算机科学的主题描述为被生成的行为. 这是可以从计算机外部观察到的行为, 例如通过屏幕或打印机. 它在与环境的交互中产生, 是计算机执行程序员编写的计算机程序中所规定的指令的结果. 计算机编程的目标在于使计算机完成特定的事情, 即生成行为. 通过执行程序, 计算机所展示的行为归根结底是由人类作为程序员所产生的.

这种行为主义观点使我们得以理解计算机科学与自然科学之间的关系: 生物学研究的是自发性行为, 而物理学则专注于无生命的自然现象, 不涉及自主行为. 生物学或物理学中系统的行为通常被描述为演化, 其中物理学中的演化是依照物理定律所发生的变换性变化. 生物学中的演化似乎缺乏内在的方向性和可预测性[174]. 这是否意味着行为在(经典)物理学中是确定性的, 而在生物学中是非确定性的? 相应类型的余代数是否能够捕捉这一情形? 就目前而言, 余代数建模理论尚未在这些领域中展现出其实用价值. 因此, 本书将重点放在数学与计算机科学中的余代数上.

行为主义观点确实有助于回答诸如计算机能思考吗?或计算机会感到疼痛吗?之类的问题. 计算机所能做的一切, 不过是展示思考的行为或疼痛的行为, 仅此而已. 但在交互中这已然足够——想想著名的图灵测试——因为归根结底, 我们永远无法确知他人是否真的感到疼痛. 我们所看到的只是疼痛行为, 并被条件化地将这种行为与某些内部状态相关联. 然而这种关联并不总是有效, 例如在不同文化中便可能失效: 在日本, 手指被烫伤后触摸耳朵是常见的反应; 而对于欧洲人来说, 这是非常规的疼痛行为. 外部行为与内部状态之间的这一问题在[350]中得到了生动的呈现——当一个Mark III Beast机器人开始伴随着相应的声音展示出拼命求生的行为时, 人类竟出乎意料地难以将其关闭, 以至于人们很容易将情感投射到机器上, 并开始对其产生怜悯之情.

这些广泛的思考构成了一套关于计算行为理论的背景, 其中可观察量与内部状态之间的关系至关重要.

我们所主张的作为计算机科学主题的被生成行为, 是由计算机依照严格的操作规则执行程序而产生的. 这种行为通常通过计算机的输入输出 (I/O) 加以观察. 更技术性地说, 程序可以被理解为一个归纳定义的项集合P中的元素.

代数和余代数

第1章 动机

本章试图解释为什么余代数是数学和计算机科学中有趣的结构. 它通过几个例子来说明这一点. 这些例子所使用的记号将在我们推进的过程中非正式地加以解释. 现阶段的重点不在于解释的精确性, 而在于思想和直觉的传递. 因此, 就目前而言, 我们将余代数——非常非正式地——定义为如下形式的函数: (1.1) S → c ⋯S⋯ (1.1) 我们的意思是: 一个余代数是由一个集合S和一个函数c给出的, 这个函数以S为定义域, 并有着一个结构性的陪域 (结果, 输出, 盒子), 在这个陪域里可能定义域S会再次出现. 这些陪域盒子的精确一般形式并不是我们当下所关心的.

一些术语: 我们经常称S为状态空间或状态集合, 并称余代数作用于S. 函数c有时被称为转换函数或转换结构. 我们将要建立的想法是余代数描述了一般性的基于状态的系统, 而其动态由函数c给出. 对于一个状态x∈S, 结果c⁡(x)告诉我们了x的后继状态是什么, 若其存在. 陪域经常被称为余代数的类型或接口. 之后我们将会看到它是一个函子.

余代数的一个简单例子是函数 ℤ → n↦ ( n−1 , n+1 ) ℤ×ℤ 其中状态空间在右边出现了两次. 因此, 这种余代数的盒子或者说类型是: (−) × (−) . 转换函数n↦(n−1,n+1)也可以使用λ记号写成λn.(n−1,n+1)或λn∈ℤ.(n−1,n+1).

这次以从ℕ到某个给定集合A的函数之集合Aℕ为状态空间, 余代数的另一个例子为 Aℕ → σ↦ ( σ⁡ (0) , λn. σ⁡ ( n+1 ) ) A× Aℕ . 这种情况下盒子是 A× (−) . 如果我们将σ写成一个无穷序列(σn)n∈ℕ, 那么我们或许可以将这个余代数写成一对函数⟨head,tail⟩, 其中 head⁡ ( ( σn ) n∈ℕ ) = σ0 而 tail⁡ ( ( σn ) n∈ℕ ) = ( σ n+1 ) n∈ℕ . 贯穿全书还会有更多的余代数的例子.

本章致力于推销余代数. 其做法在于专注于以下主题:

  1. 余代数(1.1)表示往往非常自然, 从基于状态的计算这一角度而言.
  2. 存在强大的余归纳定义以及余代数的证明原理.
  3. 存在一种非常自然(和一般)的时态逻辑与余代数相关联.
  4. 余代数概念处于合适的抽象层次上, 因而能够在各种场景下进行识别和使用.
对于最后一点的理解需要对于基本范畴论的熟悉. 这会在第1.4节提供.

评注1.0.1. 具有数学背景的读者或许会熟悉余代数作为向量空间的comonoid的概念, 对偶于代数作为monoid. 在这种情况下, 我们有一个余单位元映射V→K, 从载体空间V到潜在域K, 连带着一个余乘法V→V⊗V. 这两个映射可以组合为具有形式(1.1)的单独一个映射V→K×(V⊗V), 构成了当前意义下的余代数. 因此, 我们这里使用的余代数远比纯数学中所使用的余代数一般.

第1.1节 余代数表示的自然性

我们首先来看一个(1.1)中那样的余代数表示自然出现并且十分有用的领域, 即编程语言——用于编写计算机程序. 什么是程序, 程序又能做什么呢? 嗯, 程序是指令的列表, 告诉计算机该做什么. 说的也是. 不过, 从数学角度来看程序是什么呢? 换言之, 程序的意义是什么? 一种观点是程序是特定的函数, 其取一个输出, 然后用它来计算一个特定的结果. 这种观点并没有涵盖所有的程序: 特定的程序 (往往被称为进程) 意在永远运行下去但并不真的产生一个结果, 例如操作系统. 但是, 暂时我们将遵循将程序视为函数的观念. 我们心智中的程序不仅作用于输入上, 也作用于通常被称为状态的东西上, 例如用于存储中间结果. 程序作用于状态上的效果并不立即可见, 因而往往被称为程序的副作用. 读者可以将状态想成是由正在执行程序的计算机的内存的内容. 这并不能直接观察.

我们的程序应当能够对于状态进行修改, 典型的情况是通过像i = 5这样的赋值, 在所谓的命令式编程语言之中. 这样的一个赋值语句被解释为一个函数, 其将一个状态x转换为一个新的后继状态x′, 其中标识符i的值等于5. 也就是说, 这种编程语言中的语句是通过合适的状态转换函数描述的. 以最简单的形式来说, 忽略输入和输出, 它们将一个状态映射为一个后继状态, 如 (1.2) S → stat S (1.2) 其中我们将状态集合记为S. 它的精确结构是无关紧要的. 往往状态集合S被视为一个黑盒, 我们无法直接访问, 因而只能够观察特定的方面. 例如, 表示上述整数i函数i:S→ℤ允许我们观察i的值. 在对于赋值i = 5求值之后的结果状态x′之中值i⁡(x′)应该是5. 这个语句被考虑为一个函数S→S, 如(1.2)一般.

这种将语句视为函数S→S的描述作为最初的近似是不错的, 但是我们很快意识到语句并不总是正常终止然后产生一个后继状态. 有时它们可能会悬挂, 一直继续而不产生一个后继状态. 这种情况的发生一般是由于无限的循环, 例如在一个while语句中或者是由于没有出口的递归调用.

存在两种显然的方式来容纳这样的非终止性.

  1. 调整状态空间. 在这种情况下, 我们将状态空间S扩展为S⊥=def{⊥}∪S, 其中⊥是一个没有出现在S中的新的底元素, 特别用于指示非终止性. 那么, 语句就变成了函数: S⊥ → stat S⊥ , 要求 stat⁡ (⊥) =⊥ . 这个副条件表达了这样的想法, 一个语句悬挂了, 那么就会一直悬挂.
    这种方法的缺点在于状态空间更为复杂, 我们需要保证所有的语句都要满足这个副条件, 即它们全都保持底元素⊥. 但是, 优点在于语句复合不过就是函数复合.
  2. 调整陪域. 第二种方法原样保持状态空间S, 但是调整了语句的陪域, 如 S → stat S⊥ , 其中 S⊥ = {⊥} ∪S . 在这种呈现中, 我们可以轻易看到对于每个状态x∈S, 语句要么悬挂, 即当stat⁡(x)=⊥时, 要么正常终止, 即当stat⁡(x)=x′, 对于某个后继状态x′∈S而言. 好处在于现在不再有副条件了. 但是, 语句复合不能通过函数复合定义了, 因为类型并不匹配. 因此, 类型强制我们显式处理非终止性的传播: 对于这些种类的语句s1,s2:S→S⊥, 复合s1;s2作为函数S→S⊥是通过case distinction (或者说模式匹配) 被定义为 s1 ; s2 = λ x∈S . { ⊥ , 如果 s1 ⁡ (x) =⊥ s2 ⁡ ( x′ ) , 如果 s1 ⁡ (x) = x′ . 这个定义比函数复合更加困难 (如上述第1点所使用的), 但是它显式处理了令人感兴趣的case distinction, 即非终止和正常终止之间的交互. 因此, 被迫进行显式区别或许压根不是一件坏事.

我们可以将同样的想法再向前推进一点. 在许多编程语言 (例如Java [43]) 之中, 程序不仅可能悬挂, 也可能由于异常而意外终止. 异常发生于某个约束违反时, 例如除以零或者访问a[i]但是数组a是一个null引用. 意外终止本质上和非终止性不同: 非终止是确定的且不可撤回的, 而程序可从意外终止藉由合适的异常处理器恢复常规终止性. 在Java中, 这是由try-catch语句完成的; 见例如[43, 173, 240].

让我们记可抛出异常之集合为E. 那么, 对于可以正常终止, 意外终止或悬挂的语句, 我们又有显然的两种表示方法.

  1. 调整状态空间. 语句仍然保持为一个扩展状态空间上的自函数 (endofunction): ( {⊥} ∪S∪ ( S×E ) ) → stat ( {⊥} ∪S∪ ( S×E ) ) . 整个状态空间现在显然变得复杂起来了. 但是, 副条件也变得非平凡起来了: 我们仍然想要stat⁡(⊥)=⊥, 但是也想要stat⁡(x,e)=(x,e), 其中x∈S而e∈E, 不过后者仅对于非catch语句成立. 追踪这样的副条件或许很容易导致犯错. 不过从积极的一面来说, 语句的复合在这种表示下仍然是函数的复合.
  2. 调整陪域. 另外一种方法是将状态空间S保持原样, 但是调整语句的陪域类型, 即 (1.3) S → stat ( {⊥} ∪S∪ ( S×E ) ) . (1.3) 现在我们没有了副条件, 并且对于语句的三种可能终止模式能够作出清晰的区分. 这种结构化的输出实际上强迫我们在对于两个这样的语句s1,s2:S→({⊥}∪S∪(S×E))之复合s1;s2的定义中作出显式的区分: s1 ; s2 = λ x∈S . { ⊥ , 如果 s1 ⁡ (x) =⊥ s2 ⁡ ( x′ ) , 如果 s1 ⁡ (x) = x′ ( x′ ,e ) , 如果 s1 ⁡ (x) = ( x′ ,e ) . 因此, 如果s1悬挂或者意外终止, 那么接下来的语句s2是无需执行的. 这个事实在我们的第二种余代数表示中是非常清晰的.

当这样一种余代数表示在定理证明器的类型化语言中形式化时 (如在[265]), 定理证明器的类型检查器可以保证其作出了合适的case distinction, 根据(1.3)中的输出类型. 也见[240], 其中Java的异常机制由这样的case distinction所描述, 紧密遵循着官方语言定义[173].

这些例子刻画了作为具有结构化陪域的函数的余代数, 如(1.1), 其出现相当自然, 并且陪域指明了要施行的计算的种类. 这个想法将会进一步发展, 并应用于各种形式的计算.

第2章 多项式函子的余代数

前一章已经介绍了若干余代数的例子, 并针对这些例子阐释了行为与不变性等基本的余代数概念. 本章将深入探讨余代数领域, 引入一些基本概念, 定义和术语. 首先, 本章将以一种适当抽象的(范畴论)语言讨论若干基本的集合论构造, 如积, 余积, 指数和幂集. 这些构造被用来定义一类基本函子, 即所谓的多项式函子. 正如第2.2节将要展示的那样, 这类函子足够丰富, 能够涵盖许多有趣的余代数实例, 包括确定性自动机和非确定性自动机. 多项式函子的一个吸引人的特性在于, 几乎所有的多项式函子都具有终余代数——除非涉及(非有限的)幂集. 映入终余代数的唯一映射将作为行为态射出现, 它将每个状态映射到其行为. 本章最后两节, 即第2.4节和第2.5节, 提供了额外的背景知识, 分别涉及代数 (作为余代数的对偶) 和伴随. 伴随是一个基本的范畴论概念, 描述了贯穿数学各领域的来回转换.

第2.1节 集合上的构造

本节描述集合上的常见构造, 如积, 余积 (无交并), 指数和幂集. 这样做的目的在于统一记号, 并说明这些运算具有函子性, 即能够产生函子. 后一方面可能不太为人所熟悉. 函子性对于正确发展余代数理论至关重要; 参见定义1.4.5.
